Findings by severity

Severity = accessibility/usability impact. critical blocks AT users or breaks semantics · serious WCAG AA fail · moderate friction · low polish.

critical  1. Tabs reference a tab-panel that doesn't exist

axe: aria-valid-attr-value · all 4 tabs (Today/Trends/Body/Train) · every surface

Each tab button has aria-controls="v2-tabpanel-{name}" but no element with that id (or role="tabpanel") is rendered. Screen readers announce a broken relationship.

Fix: give the tab content container id="v2-tabpanel-{name}" + role="tabpanel" + aria-labelledby back to the tab, or remove aria-controls if the panel can't be marked. See tab-bar.component.ts.

critical  2. History calendar grid is missing its row/cell roles

axe: aria-required-children (1× div[role="grid"]) + aria-required-parent (42 day buttons) · History

The calendar uses role="grid" with day-<button>s as direct children — no role="row" wrappers or role="gridcell". ARIA grid semantics are invalid, so grid navigation/announcements break.

Fix: wrap each week in role="row" and each day in role="gridcell", or drop the grid roles entirely (a labelled list of day buttons is valid and simpler). See history.component.ts.

critical  3. CSV-import file input has no accessible name

axe: label · Settings → Data

<input type="file" class="sr-only"> has no label/aria-label/title. Low real-world impact (a visible labelled button triggers it), but it's a flagged AA failure.

Fix: add aria-label="Import CSV file" to the hidden input. See settings-data-section.component.ts.

serious  4. Muted text color fails AA contrast — app-wide

axe: color-contrast · 9–10 nodes per surface · captions and form-field labels

--v2-ink-muted: #7a7268 measures 4.43:1 on the light background (#faf7f2) — just under the 4.5:1 AA minimum — and only 3.45:1 on the accent-soft card tint (#f3d6cc). It's used on captions, the "/ 120g" goal text, day-cell numbers, and field labels like "Calories *" / "Protein (g)".

Fix: darken one token. Changing --v2-ink-muted to ≈#6e665b (~5:1 on paper) clears AA everywhere; also re-check the dark-variant #847a68. Single edit in styles-v2.css:26.

serious  5. Progress-ring SVG lacks a text alternative

axe: svg-img-alt · .v2-ring__svg · History (and any ring rendered without a label)

The calorie ring SVG has an image role but no <title>/aria-label in this context, so its value isn't announced. (On Today the ring does carry "Calories: 0 of 2000" — so the fix is to make that consistent.)

Fix: always pass an aria-label/<title> to the ring component (e.g. "{day}: {kcal} of {goal}"). See the shared ring used by history/day-summary.

moderate  6. Modals don't move focus inside on open (no focus trap)

heuristic: dialog.contains(activeElement) === false · Add-food, Settings (all ui-sheets)

When a sheet opens, keyboard focus stays on the trigger button behind the overlay. Keyboard/AT users must tab through the whole page to reach the dialog, and focus can escape behind it. Esc-to-close already works.

Fix: on open, move focus to the dialog (or its close/first field), trap Tab within it, and restore focus to the trigger on close. Centralize in ui/sheet.component.ts so every sheet inherits it.

moderate  7. Touch targets below 44×44 — pervasive

heuristic · 7–47 sub-44px interactives per surface

The small-button style .v2-btn--sm renders ~36px tall (New, Start, Manual/Search/Photo/Barcode, water +250/+500/+1L, language, sign-out, settings toggles). Inline text-buttons are 16–17px ("manage", "Started earlier?", footer privacy/terms/status/contact). Calendar day cells are 41px. WCAG 2.5.5 / platform guidance want ≥44px.

Fix: raise .v2-btn--sm min-height to 44px (keep visual size via padding if needed) and give inline text-buttons a larger hit area (padding / min-height). Space the footer links apart.

low  8. Desktop layout leaves the screen mostly empty

visual · ≥1024px · Today, Train, etc.

Content is a single narrow column; on wide screens there's large dead space and the floating + FAB sits far to the right, detached from the column. Acceptable for a mobile-first PWA but feels unfinished on desktop.

Fix (optional): center the column with a comfortable max-width, anchor the FAB to the column, or use a 2-column layout on ≥1024px.

low  9. History: washed-out days & empty space below calendar

visual · History

Future / other-month day numbers and the S-M-T-W-T-F-S weekday headers are near-invisible against cream. Below the calendar is a large blank gap with no selected-day preview or empty state.

Fix (optional): nudge disabled-day and weekday-header contrast up a touch; consider a selected-day summary / recent-entries strip below the grid.

low  10. Footer link cluster is tiny and tightly packed

visual + touch · every surface

privacy · terms · status · contact render at ~17px tall with small gaps — easy to mis-tap on mobile.

Fix (optional): increase tap padding and spacing; covered by fix #7.
